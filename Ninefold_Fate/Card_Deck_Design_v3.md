# 五英雄卡組設計 v3（修訂版）

> 根據以下問題全面修訂：
> - 水系倒數機制重新定性（負面效果 + 傷害補償）
> - 光系移除抽牌機制，改為護盾體系
> - 移除所有卡組內重複圖塊
> - 圖塊效果改以 AbilityComponent ID 對應 GDScript 類別
> - 火系簡化為純傷害
> - 草系回血加入英雄攻擊力條件與 HP 觸發條件
> - 暗系新增扣血換攻機制

---

## 程式架構對照

JSON 中的 `special_effects: Array[String]` 對應 `AbilityComponent.create_by_id()`：

```
現有 ability ID         GDScript 類別
─────────────────────── ─────────────────────
heal                 →  HealAbility
shield               →  ShieldAbility
freeze               →  FreezeAbility
debuff_defense       →  DefenseDebuffAbility

待新增
─────────────────────── ─────────────────────
damage_multiplier    →  DamageMultiplierAbility
countdown_push       →  CountdownPushAbility
lifesteal            →  LifestealAbility
lifesteal_amplify    →  LifestealAmplifyAbility
aoe_damage           →  AoeDamageAbility
crit_boost           →  CritBoostAbility
hp_sacrifice         →  HpSacrificeAbility
```

## 二、🔵 霜凜的激流牌組（風險對賭型）

### 機制核心：countdown_push 是負面效果

```
⚠️  countdown_push 的正確理解：
    每次放置水系卡 → 敵人倒數額外 -N
    倒數更快歸零 → 敵人更早攻擊 → 對玩家不利

    但補償：水系卡附帶更高的傷害倍率
    玩法邏輯：「我讓你更快打我，但我先把你打殘！」

    Freeze (B110) 是安全閥：
    敵人倒數歸零這一次不攻擊，讓玩家有機會打完高風險回合
```

### 8 張卡（無重複）

| ID   | 屬性 | 形狀    | 稀有度    | 效果                           |
|------|------|--------|----------|--------------------------------|
| B002 | 水   | single | common   | 倒數 -1（敵人快攻！）+ 傷害 ×1.2  |
| B108 | 水   | line_2 | common   | 倒數 -1 + 傷害 ×1.3              |
| B102 | 水   | line_3 | uncommon | 倒數 -2 + 傷害 ×1.5（高賭注）    |
| B109 | 水   | L型    | uncommon | 倒數 -1 + 傷害 ×1.4              |
| B110 | 水   | square | uncommon | Freeze：倒數歸零不攻擊（防禦版）  |
| B005 | 暗   | single | common   | 暴擊率 +20% / ×1.5（殺傷力加成） |
| B114 | 光   | T型    | uncommon | 回血 40（被打後的回復）          |
| B003 | 草   | single | common   | Lifesteal 10% hero_atk         |

**策略要點：**
- 計算敵人何時倒數歸零，決定何時押注 B102（最大傷害）
- Freeze (B110) 是保命牌，不要亂用，留給倒數快歸零的關鍵回合
- 敵人倒數充裕時可以多放幾張積累傷害；倒數快到時切換策略
- 搭配 B005 暴擊在高倍率水系卡觸發，單回合傷害可達到破千

**危險情境示範：**
```
敵人倒數：3 → 我方放 B102（-2）→ 敵人倒數變 1 ！
下回合敵人倒數 -1（正常）→ 歸零 → 敵人攻擊

此時應在倒數 = 1 那回合放 B110（Freeze）防禦
或者在倒數 = 3 時就放高傷害組合，確保那一回合把敵人打死
```

## 三、🟢 綠葉的循環牌組（持久戰型）

### 機制核心：Lifesteal 基於英雄攻擊力

```
設計原則：lifesteal 不依賴造成的傷害值
         而是依賴 hero.base_attack × percent

理由：
  若 lifesteal = 傷害% → 弱敵存在時回血幾乎為零（傷害低）
                          強敵存在時傷害高、回血也高，結果無聊
  若 lifesteal = hero_atk% → 回血固定可預測
                               隨英雄成長而提升，不受敵人強弱影響
                               玩家知道「我每回合大約能回多少血」
```

### 8 張卡（無重複）

| ID   | 屬性 | 形狀    | 稀有度    | 效果                                     |
|------|------|--------|----------|------------------------------------------|
| B003 | 草   | single | common   | Lifesteal 10% hero_atk（無條件）          |
| B111 | 草   | line_2 | common   | 直接回血 30（固定，應急）                  |
| B103 | 草   | T型    | uncommon | Lifesteal 15% hero_atk（條件：HP < 70%）  |
| B112 | 草   | line_3 | uncommon | 本回合所有 lifesteal 倍率 ×1.25           |
| B203 | 草   | cross  | rare     | Lifesteal 20% hero_atk + 直接回血 50     |
| B004 | 光   | single | common   | 護盾 15（主動扛傷，非只靠回血）            |
| B002 | 水   | single | common   | 倒數 -1 + 傷害 ×1.2（加快戰鬥節奏）       |
| B005 | 暗   | single | common   | 暴擊率 +20% / ×1.5（提升傷害）            |

**策略要點：**
- B103 的 HP < 70% 條件：鼓勵玩家「讓自己保持在危險血量以觸發高效吸血」
- B112 倍率卡：單獨放沒用，必須和其他 lifesteal 卡配合，教玩家規劃出牌順序
- B111 固定回血：不依賴任何條件，用於 HP 告急的純應急場合
- 草系與暗系的差別：草是穩定的漸進回血；暗是先扣後補的高風險操作

**回血量計算示例（英雄 ATK = 90）：**
```
B003: 90 × 10% = 9 HP / 回合
B103: 90 × 15% = 13.5 HP（HP < 70% 時）
B112 + B003: 9 × 1.25 = 11.25 HP（有倍率加成）
B203: 90 × 20% + 50 = 68 HP（最高單卡回血）
```

## 四、🟡 晨曦的護盾牌組（全能型）

### 機制核心：護盾 + AoE（移除原抽牌機制）

```
移除「抽牌加速」的原因：
  每回合確認後自動補回使用掉的牌
  若額外抽牌 → 手牌 > 4 → 手牌無限堆積
  → 此機制在目前架構下不可實裝

改為護盾體系：
  護盾 (ShieldAbility) 已在 Godot 中實裝
  可以疊加，作為預防傷害的主動機制
```

### 8 張卡（無重複）

| ID   | 屬性 | 形狀    | 稀有度    | 效果                          |
|------|------|--------|----------|-------------------------------|
| B004 | 光   | single | common   | 護盾 15                       |
| B113 | 光   | line_2 | common   | 護盾 25                       |
| B201 | 光   | cross  | rare     | AoE ×1.2 打全敵 + 護盾 40     |
| B114 | 光   | T型    | uncommon | 回血 40                       |
| B115 | 光   | L型    | uncommon | 傷害 ×1.15 + 護盾 20           |
| B003 | 草   | single | common   | Lifesteal 10% hero_atk        |
| B102 | 水   | line_3 | uncommon | 倒數 -2 + 傷害 ×1.5（冒進選項）|
| B005 | 暗   | single | common   | 暴擊率 +20% / ×1.5             |

**策略要點：**
- 護盾疊加：B004 + B113 在一個回合放完可以堆 40 護盾，很抗打
- B201 十字聖光是光系核心：多敵場景 AoE + 護盾 40，攻防同時
- B102 水系外援：護盾足夠時可以冒進，犧牲倒數換高傷
- B114 是純治療，護盾破了之後的補充手段

**護盾疊加示範：**
```
回合 1：放 B113（護盾 25）+ B004（護盾 15）→ 護盾 40
回合 2：放 B201（護盾 40）→ 護盾共 80（假設上回合沒被打）
回合 3：放 B115（護盾 20）→ 護盾共 100，幾乎無敵狀態！
```

## 五、⚫ 夜幕的扣血牌組（操作性最強）

### 機制核心：HP 作為資源消耗

```
設計邏輯：
  hp_sacrifice 讓 HP 不只是「生存計量」
  也是「進攻資源」
  玩家需要決策：現在 HP 夠用來扣血換傷害嗎？

  vs 草系的差別：
  草系：先打傷害，然後回血（被動保護）
  暗系：先決定扣血，然後造成高傷，然後吸血回收（主動消耗+回收）

  操作性體現在：
  HP > 100 才能扣大血（B104）
  HP > 80 才能扣小血（B117）
  HP 低時切換：放護盾或回血卡保命
```

### 8 張卡（無重複）

| ID   | 屬性 | 形狀    | 稀有度    | 效果                                     |
|------|------|--------|----------|------------------------------------------|
| B005 | 暗   | single | common   | 暴擊率 +20%，暴擊倍率 ×1.5               |
| B116 | 暗   | line_2 | common   | 敵方防禦 -15%，持續 1 回合                |
| B104 | 暗   | square | uncommon | 扣 50HP → 傷害 ×1.5（HP < 100 自動跳過）  |
| B117 | 暗   | S型    | uncommon | 扣 30HP → 傷害 ×1.3 + lifesteal 15% hero_atk |
| B118 | 暗   | line_3 | uncommon | 暴擊率 +30%，暴擊倍率 ×1.8（暴擊頂點）    |
| B003 | 草   | single | common   | Lifesteal 10% hero_atk（扣血後主要回收）  |
| B111 | 草   | line_2 | common   | 直接回血 30（HP 太低時的應急牌）          |
| B004 | 光   | single | common   | 護盾 15（HP 不足以扣血時的過渡防禦）       |

**策略要點：**
- 標準操作順序：B116（削弱） → B104（扣血換傷） → B005/B118（暴擊放大） → B003（吸血回收）
- 風險管理：HP > 200 時大膽扣血；HP 在 100~200 之間謹慎；HP < 100 切換防守
- B117 扣血 + 吸血是自我平衡設計：犧牲 30HP 然後透過傷害回收部分

**HP 節奏示例（英雄 HP 800，ATK 120）：**
```
回合 1：HP 800，放 B116（削弱） + B104（-50HP = 750）
       傷害：120 + bonus × 1.5 = 高輸出
       
回合 2：HP 750，放 B005（暴擊）+ B117（-30HP = 720）
       傷害 ×1.3，若暴擊 ×1.5 疊算
       吸血：120 × 15% = 18HP 回收 → HP ≈ 738
       
回合 3：HP 738，放 B003（吸血）+ B118（暴擊 +30%）
       繼續輸出，吸血 12HP
       
若 HP 降至 100 以下：改放 B111 回血 30 + B004 護盾 15
再等 HP 回到安全值再重啟扣血操作
```

## 七、新增 AbilityComponent 實裝優先級

```
優先實裝（影響核心機制）：
1. damage_multiplier  → 火、水、光系全用
2. countdown_push     → 水系機制核心
3. lifesteal          → 草系、暗系都需要
4. crit_boost         → 暗系核心

次優先（強化體驗）：
5. hp_sacrifice       → 暗系特色機制
6. lifesteal_amplify  → 草系倍率牌
7. aoe_damage         → 火系稀有牌、光系稀有牌

延後（可用現有系統模擬）：
8. 可以用現有 damage_dealt 事件 + BattleManager 邏輯處理 aoe
```
